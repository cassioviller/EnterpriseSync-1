## Passo 2: Função Geradora de Lançamentos Contábeis
No arquivo `contabilidade_utils.py`, crie uma função universal para gerar os lançamentos duplos (Partidas Dobradas):
```python
def gerar_lancamento_contabil_automatico(admin_id, tipo_operacao, valor, data, descricao, centro_custo_id=None):
    """Gera o lançamento contábil de débito e crédito automaticamente"""
    from models import db, LancamentoContabil, PartidaContabil
    
    mapeamento = MAPEAMENTO_CONTABIL.get(tipo_operacao)
    if not mapeamento:
        return False
        
    # 1. Cria o cabeçalho do lançamento
    lancamento = LancamentoContabil(
        numero=get_next_lancamento_numero(admin_id),
        data_lancamento=data,
        descricao=descricao,
        tipo='AUTOMATICO',
        admin_id=admin_id
    )
    db.session.add(lancamento)
    db.session.flush() # Para obter o ID
    
    # 2. Partida de Débito
    debito = PartidaContabil(
        lancamento_id=lancamento.id,
        conta_codigo=mapeamento['debito'],
        centro_custo_id=centro_custo_id,
        tipo_partida='DEBITO',
        valor=valor,
        admin_id=admin_id
    )
    
    # 3. Partida de Crédito
    credito = PartidaContabil(
        lancamento_id=lancamento.id,
        conta_codigo=mapeamento['credito'],
        centro_custo_id=centro_custo_id,
        tipo_partida='CREDITO',
        valor=valor,
        admin_id=admin_id
    )
    
    db.session.add(debito)
    db.session.add(credito)
    
    # 4. Atualiza o Balancete Mensal
    atualizar_balancete_mensal(admin_id, mapeamento['debito'], data, valor, 'DEBITO')
    atualizar_balancete_mensal(admin_id, mapeamento['credito'], data, valor, 'CREDITO')
    
    return True
```

## Passo 3: Injeção dos Gatilhos (Triggers)
Agora, injete a chamada dessa função nos pontos chave do sistema (apenas se `is_v2_active()` for True):
1. **Em `compras_views.py` (ou `almoxarifado_views.py`):** Ao salvar uma nova compra, chame `gerar_lancamento_contabil_automatico(..., 'compra_material', ...)`.
2. **Em `alimentacao_views.py`:** Ao salvar o lançamento de marmitas, chame com `'despesa_alimentacao'`.
3. **Em `transporte_views.py`:** Ao salvar o lançamento de transporte, chame com `'despesa_transporte'`.
4. **Em `folha_pagamento_views.py`:** Ao fechar a folha do mês, chame com `'folha_pagamento'`.

## Passo 4: Integração com o Fluxo de Caixa (Baixas)
No arquivo `financeiro_views.py`, localize a função que dá "Baixa" (pagamento) em uma `ContaPagar`.
1. Quando uma conta a pagar for marcada como 'PAGO', verifique a origem dela.
2. Se for origem de compra de material, chame `gerar_lancamento_contabil_automatico(..., 'pagamento_fornecedor', ...)`.
3. Se for origem de folha, chame `'pagamento_salario'`.
4. Isso garante que o passivo (Conta a Pagar) seja zerado e o dinheiro saia do Ativo (Banco/Caixa).
