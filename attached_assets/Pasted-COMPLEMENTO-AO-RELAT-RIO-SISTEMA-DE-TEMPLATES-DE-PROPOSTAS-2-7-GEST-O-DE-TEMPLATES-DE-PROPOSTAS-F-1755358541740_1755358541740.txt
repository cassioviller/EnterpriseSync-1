2.7 GESTÃO DE TEMPLATES DE PROPOSTAS
FUNCIONALIDADES DO SISTEMA DE TEMPLATES
Criação e Gestão de Templates (CRUD)
Criar templates personalizados com nome e descrição
Definir itens padrão para cada template
Controle de status (ativo/inativo)
Organização por categoria de projeto
Versionamento com data de criação
Exclusão lógica (manter histórico)
Estrutura de Templates
proposal_templates:
- id (identificador único)
- name (nome do template)
- description (descrição/uso)
- defaultItems (JSON com itens padrão)
- itemsPadrao (compatibilidade)
- active (ativo/inativo)
- adminId (proprietário)
- createdAt (data de criação)
Aplicação de Templates
Seleção rápida durante criação de proposta
Carregamento automático de itens pré-definidos
Personalização posterior dos itens carregados
Múltiplos templates por usuário/empresa
Preview da quantidade de itens do template
Exemplos de Templates Implementados
Template 1 - MEZANINO:

Estrutura para projetos de mezanino
Itens específicos: vigas, pilares, escadas
Unidades padrão: metro, tonelada, metro²
Template 2 - ESTRUTURA METÁLICA:

Projetos de estruturas industriais
Itens: vigas W, soldas, parafusos
Cálculos específicos de peso e área
Template 3 - COBERTURA ÁREA GOURMET:

Coberturas residenciais
Itens: telhas, estrutura, acabamentos
Medidas em metro² e unidades
APIs do Sistema de Templates
Backend (server/routes.ts)
// Listar templates ativos
GET /api/proposal-templates
- Retorna templates do usuário logado
- Filtro por status ativo
- Ordenação por data de criação
// Criar novo template
POST /api/proposal-templates
- Validação de dados obrigatórios
- Associação ao usuário logado
- Salvamento dos itens padrão
