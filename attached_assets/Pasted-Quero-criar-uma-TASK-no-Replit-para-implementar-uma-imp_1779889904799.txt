Quero criar uma TASK no Replit para implementar uma importação segura de fluxo de caixa no sistema EnterpriseSync.

IMPORTANTE:
Não implemente direto sem antes criar a task, analisar o projeto e montar um plano.
Essa task depois será revisada pelo Claude Code antes da execução final.

Contexto do projeto:
Tenho um sistema chamado EnterpriseSync rodando no Replit/GitHub, com banco PostgreSQL hospedado no EasyPanel.

O sistema já possui algum modo de importação. Antes de criar qualquer script novo, você deve estudar a lógica atual de importação do projeto para reaproveitar o padrão existente e evitar criar uma lógica paralela.

Objetivo da task:
Criar um script seguro para importar um arquivo Excel de fluxo de caixa para dentro do banco de dados do EnterpriseSync, populando corretamente as movimentações financeiras no perfil/admin correto.

O perfil será definido por `admin_id`.

Exemplo atual:
admin_id = 1640
empresa/perfil = VEKS

Mas o script não deve ficar fixo somente nesse admin_id. Ele deve aceitar `admin_id` como parâmetro.

Arquivos e informações que posso fornecer depois:
- arquivo Excel do fluxo de caixa;
- URL de conexão do PostgreSQL do EasyPanel;
- prints do banco;
- prints das categorias já cadastradas;
- nomes das tabelas;
- exemplos de lançamentos corretos;
- documentos adicionais.

Você pode e deve fazer perguntas antes de implementar caso falte alguma informação.
Não invente regras críticas.

Se tiver dúvida sobre categoria, centro de custo, cliente, fornecedor, banco, forma de pagamento, status, duplicidade ou tabela final de destino, pergunte antes.

A task deve seguir este fluxo obrigatório:

1. Diagnóstico do projeto
- Identificar qual stack o projeto usa.
- Encontrar os modelos/tabelas do banco.
- Encontrar a lógica atual de importação.
- Identificar quais tabelas armazenam:
  - movimentações financeiras;
  - entradas;
  - saídas;
  - categorias/plano de contas;
  - centros de custo;
  - clientes;
  - fornecedores;
  - contas bancárias;
  - formas de pagamento;
  - status de pagamento.
- Entender como o sistema separa entrada e saída.

2. Plano antes de alterar arquivos
Antes de alterar qualquer arquivo, apresentar:
- diagnóstico do que foi encontrado;
- tabelas envolvidas;
- campos obrigatórios;
- riscos da importação;
- perguntas pendentes;
- plano de implementação.

3. Script de importação
Criar um script, por exemplo:

scripts/importar_fluxo_caixa.py

O script deve aceitar parâmetros:

--file "arquivo.xlsx"
--admin-id 1640
--dry-run
--commit

Regras obrigatórias:
- `--dry-run` deve simular a importação sem gravar nada no banco.
- `--commit` só deve gravar no banco depois de autorização explícita.
- Nunca rodar `--commit` automaticamente.
- Não colocar senha, URL do banco ou credenciais fixas no código.
- Usar variável de ambiente, preferencialmente `DATABASE_URL`.
- Se a URL interna do EasyPanel não funcionar no Replit, explicar o motivo e pedir alternativa.

4. Leitura do Excel
O script deve ler o arquivo Excel e identificar abas como:
- Entrada;
- Saída;
- Cadastros;
- Cadastros 2;
- Clientes e Fornecedores;
- outras abas existentes.

A aba Entrada deve ser importada como receita/entrada.
A aba Saída deve ser importada como despesa/saída.

5. Mapeamento de campos
Para cada linha do Excel, tentar mapear:

- data;
- tipo: entrada ou saída;
- plano de contas/categoria;
- cliente ou fornecedor;
- descrição;
- centro de custo;
- banco/conta;
- forma de pagamento;
- parcela;
- nota fiscal/fatura;
- status;
- valor;
- DRE, se existir;
- admin_id.

6. Categorias existentes
O script deve usar as categorias já existentes no banco para o `admin_id` informado.

Ele deve tentar casar o texto do Excel com as categorias do banco.

Se não encontrar correspondência segura, não deve inventar categoria.
Nesse caso, deve marcar como “não classificado” no relatório.

7. Duplicidades
Antes de inserir, o script deve verificar se já existe lançamento parecido com:

- mesma data;
- mesmo valor;
- mesma descrição;
- mesmo tipo;
- mesmo admin_id;
- mesmo cliente/fornecedor, quando possível.

8. Prévia antes da importação
O script deve gerar uma prévia em CSV ou JSON antes de qualquer importação.

Essa prévia deve conter:

- linha original do Excel;
- aba de origem;
- data;
- tipo;
- descrição;
- valor;
- categoria encontrada;
- centro de custo encontrado;
- cliente/fornecedor encontrado;
- banco/conta encontrado;
- forma de pagamento encontrada;
- status;
- ação sugerida: importar, ignorar, duplicado, erro ou não classificado;
- motivo do erro ou alerta.

9. Relatório final
O script deve gerar relatório final com:

- total de linhas lidas;
- total de entradas;
- total de saídas;
- total importado;
- total ignorado;
- total duplicado;
- total não classificado;
- total com erro;
- soma de valores importados por tipo;
- lista dos registros que precisam de revisão manual.

10. Banco de dados
Não alterar estrutura do banco sem autorização.
Se precisar criar tabela, campo, índice ou migration, explicar antes e pedir aprovação.

11. Reaproveitamento
Se já existir importador atual no EnterpriseSync, reaproveitar a lógica existente sempre que fizer sentido.
Não criar uma lógica paralela que quebre o padrão do projeto.

12. Entregáveis esperados da task
A task deve produzir:

- diagnóstico técnico;

- plano de implementação;
- script de importação com dry-run e commit;
