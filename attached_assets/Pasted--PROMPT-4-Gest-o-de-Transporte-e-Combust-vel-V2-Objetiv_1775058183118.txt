# PROMPT 4: Gestão de Transporte e Combustível (V2)

**Objetivo:** Criar um módulo integrado para gerenciar despesas de transporte (Vale Transporte, Combustível, Passagens) vinculando-as diretamente aos funcionários/veículos e aos centros de custo das obras. Integrar automaticamente com a nova Gestão de Custos (Prompt 2).

**Arquivos a serem modificados:**
- `models.py`
- `transporte_views.py`
- `templates/transporte/novo_lancamento.html`
- `utils/financeiro_integration.py` (chamada da função `registrar_custo_automatico`)

Por favor, implemente as seguintes alterações passo a passo:

## Passo 1: Modelagem de Dados de Transporte
No arquivo `models.py`, crie ou adapte os modelos para gerenciar os lançamentos de transporte de forma genérica:

```python
class CategoriaTransporte(db.Model):
    __tablename__ = 'categoria_transporte'
    id = db.Column(db.Integer, primary_key=True)
    nome = db.Column(db.String(50), nullable=False) # 'Vale Transporte', 'Combustível', 'Aplicativo', 'Passagem Aérea'
    admin_id = db.Column(db.Integer, db.ForeignKey('usuario.id'), nullable=False)

class LancamentoTransporte(db.Model):
    __tablename__ = 'lancamento_transporte'
    id = db.Column(db.Integer, primary_key=True)
    categoria_id = db.Column(db.Integer, db.ForeignKey('categoria_transporte.id'), nullable=False)
    funcionario_id = db.Column(db.Integer, db.ForeignKey('funcionario.id'), nullable=True)
    veiculo_id = db.Column(db.Integer, db.ForeignKey('frota_veiculo.id'), nullable=True)
    centro_custo_id = db.Column(db.Integer, db.ForeignKey('centro_custo.id'), nullable=False)
    obra_id = db.Column(db.Integer, db.ForeignKey('obra.id'), nullable=True)
    
    data_lancamento = db.Column(db.Date, nullable=False)
    valor = db.Column(db.Numeric(10, 2), nullable=False)
    descricao = db.Column(db.String(200))
    comprovante_url = db.Column(db.String(500)) # Caminho do anexo
    
    admin_id = db.Column(db.Integer, db.ForeignKey('usuario.id'), nullable=False)
    created_at = db.Column(db.DateTime, default=datetime.utcnow)
    
    # Relacionamentos
    categoria = db.relationship('CategoriaTransporte')
    funcionario = db.relationship('Funcionario')
    veiculo = db.relationship('Vehicle')
    centro_custo = db.relationship('CentroCusto')
```

## Passo 2: Interface de Lançamento de Transporte
Crie uma nova tela (`templates/transporte/novo_lancamento.html`):
1. Formulário com os campos: Categoria, Data, Valor, Descrição e Anexo.
2. Lógica condicional na UI (JavaScript):
   - Se a categoria for "Combustível", mostre o campo "Veículo" (select com a frota).
   - Se a categoria for "Vale Transporte" ou "Aplicativo", mostre o campo "Funcionário".
3. Campo obrigatório "Centro de Custo" para apropriação correta da despesa.

## Passo 3: Processamento no Backend e Integração com Gestão de Custos
Crie as rotas em `transporte_views.py`:
1. Rota POST para salvar o `LancamentoTransporte`.
2. Upload de arquivo: Salve o comprovante na pasta `static/uploads/comprovantes/` e grave o caminho no banco.
3. **Integração Crucial:** Após salvar o lançamento, chame a função `registrar_custo_automatico` (criada no Prompt 2):
   - `tipo_categoria = 'TRANSPORTE'`
   - `entidade_nome = veiculo.placa` (se for combustível) ou `funcionario.nome` (se for VT/App)
   - `entidade_id = veiculo.id` ou `funcionario.id`
   - `data = lancamento.data_lancamento`
   - `descricao = f"{categoria.nome} - {lancamento.descricao}"`
   - `valor = lancamento.valor`
   - `obra_id = lancamento.obra_id`
   - `origem_tabela = 'lancamento_transporte'`
   - `origem_id = lancamento.id`
4. Isso garante que, na tela de Gestão de Custos, apareça um Registro Pai "Despesa Transporte - Veículo ABC-1234" com o valor total, e ao expandir, mostre os abastecimentos.

## Passo 4: Proteção V1/V2
No menu lateral (`templates/base.html`), adicione o link para "Gestão de Transporte" dentro do menu "Financeiro" ou "Frota", mas envolva-o com `{% if is_v2_active() %}` para que apenas clientes V2 vejam esta nova funcionalidade. Clientes V1 continuarão usando o lançamento genérico de despesas.
