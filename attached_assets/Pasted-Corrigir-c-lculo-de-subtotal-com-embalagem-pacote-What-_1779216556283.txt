Corrigir cálculo de subtotal com embalagem/pacote
What & Why
When a supply item has a fator_comercial (packaging multiple) greater than 1, the unit price represents the price of a package, not a single unit. The current code correctly rounds up the quantity to the next package multiple, but then incorrectly multiplies the full technical quantity by the unit price instead of multiplying the number of packages by the package price.

Example (wrong today):

Screw, coef=14.8148, service qty=100, fator=100, price=R$46.00
qtd_tecnica = 1481.48, qtd_compra = 1500, subtotal = 1500 × 46 = R$69,000 ❌
Correct result:

qtd_pacotes = ceil(1481.48 / 100) = 15, subtotal = 15 × 46 = R$690 ✅
Done looks like
In the budget editor screen, the "Subtotal (compra)" column shows the correct value for items with fator_comercial > 1 (price × number of packages, not price × total units).
The value shown live in the JS preview matches what gets saved to the database.
Items with fator_comercial = 1 are unaffected (behaviour unchanged).
Out of scope
Changing how preco_unitario is labeled or entered (no UI label changes).
Fixing the unidade field on catalogue items (mentioned in the attachment but a separate catalogue-data concern).
Any other budget calculation logic (margins, taxes, pricing).
Steps
Fix backend calculation — In services/orcamento_view_service.py, inside the fator > 1 branch, replace sub_compra = qtd_com * preco with sub_compra = multiplos * preco. The variable multiplos (number of packages, already computed on the line above) is the correct multiplier. Also update the inline comment on line 79 to reflect the corrected formula.

Fix Jinja template subtotal cell — In templates/orcamentos/editar.html, line 553, the expression (_qtd_com * _preco) should become (((_qtd_tec / _fator)|round(0, 'ceil')) * _preco) if _fator > 1 else (_qtd_tec * _preco) so the rendered-on-load value matches the corrected logic. _qtd_tec and _fator are already computed just above that line.

Fix JavaScript live-preview calculation — In the same file, in the recalc() JS function (~line 843), add a qtdPacotes variable (fator > 1 ? Math.ceil(qtdTec / fator) : qtdTec) and change const subTot = qtdCom * preco to const subTot = qtdPacotes * preco. The display of qtdCom (total technical units bought) stays unchanged — only the subtotal multiplier changes.
