## 1. VISÃO GERAL DO MÓDULO

### 1.1 PROPÓSITO
O **Módulo de Almoxarifado** é um sistema completo de gestão de materiais, ferramentas e EPIs para empresas de construção civil. Ele controla entrada, saída, devolução e estoque de itens, com rastreabilidade total e integração nativa ao sistema SIGE.

### 1.2 OBJETIVOS PRINCIPAIS
- **Controlar estoque** de materiais, ferramentas e EPIs
- **Rastrear movimentações** (entrada, saída, devolução)
- **Gerenciar responsabilidades** (quem está com cada item)
- **Controlar custos** por obra/centro de custo
- **Alertar sobre** estoque baixo e validades
- **Gerar relatórios** essenciais para gestão

### 1.3 USUÁRIOS ALVO
- **Almoxarife:** Controla entrada/saída de materiais
- **Encarregado:** Solicita materiais para obras
- **Gestor:** Acompanha custos e relatórios
- **Funcionário:** Visualiza itens sob sua responsabilidade

## 2. FUNCIONALIDADES DETALHADAS

### 2.1 GESTÃO DE CATÁLOGO

#### 2.1.1 Categorias de Materiais
**Funcionalidade:** Organizar itens em categorias com configurações padrão.

**Categorias Padrão:**
1. **EPIs** (Equipamentos de Proteção Individual)
   - Tipo padrão: SERIALIZADO
   - Permite devolução: SIM
   - Exemplos: Capacetes, óculos, botas

2. **Ferramentas**
   - Tipo padrão: SERIALIZADO
   - Permite devolução: SIM
   - Exemplos: Furadeiras, martelos, chaves

3. **Materiais Elétricos**
   - Tipo padrão: CONSUMÍVEL
   - Permite devolução: SIM (sobras)
   - Exemplos: Fios, cabos, disjuntores

4. **Materiais Hidráulicos**
   - Tipo padrão: CONSUMÍVEL
   - Permite devolução: SIM (sobras)
   - Exemplos: Tubos, conexões, registros

5. **Consumíveis Descartáveis**
   - Tipo padrão: CONSUMÍVEL
   - Permite devolução: NÃO
   - Exemplos: Luvas, máscaras, eletrodos

#### 2.1.2 Cadastro de Itens
**Funcionalidade:** Cadastrar itens do almoxarifado com suas especificações.

**Tipos de Controle:**
- **SERIALIZADO:** Itens únicos com número de série (sempre podem ser devolvidos)
- **CONSUMÍVEL:** Materiais por quantidade (devolução configurável)

**Configuração de Devolução:**
- **Serializados:** Sempre `permite_devolucao = True` (não editável)
- **Consumíveis:** Configurável por item (`True` para sobras, `False` para descartáveis)

### 2.2 MOVIMENTAÇÕES

#### 2.2.1 Entrada de Materiais
**Funcionalidade:** Registrar entrada de novos materiais no estoque.

**Fluxos:**
- **Itens Serializados:** Informar números de série individuais
- **Itens Consumíveis:** Informar quantidade total

#### 2.2.2 Saída de Materiais
**Funcionalidade:** Registrar retirada de materiais por funcionários.

**Fluxos:**
- **Itens Serializados:** Selecionar itens específicos por número de série
- **Itens Consumíveis:** Informar quantidade a retirar

#### 2.2.3 Devolução Inteligente
**Funcionalidade:** Processar devolução baseada no tipo de item.

**Fluxos:**
- **Serializados:** Devolução obrigatória do item específico
- **Consumíveis Retornáveis:** Devolução opcional de sobras
- **Consumíveis Descartáveis:** Não permite devolução

### 2.3 CONTROLE E RELATÓRIOS

#### 2.3.1 Dashboard Executivo
**Funcionalidade:** Visão geral do almoxarifado com KPIs essenciais.

#### 2.3.2 Relatórios Essenciais
**Funcionalidade:** 5 relatórios principais para gestão.

## 4. FLUXO DE DADOS DETALHADO

### 4.1 FLUXO DE ENTRADA

```
1. Usuário acessa /almoxarifado/entrada
2. Seleciona item → AJAX carrega tipo de controle
3. Preenche dados gerais (obra, valor, data)
4. SE SERIALIZADO:
   - Informa números de série (textarea)
   - Sistema valida unicidade
5. SE CONSUMÍVEL:
   - Informa quantidade
   - Sistema valida > 0
6. Submete formulário → POST /almoxarifado/processar-entrada
7. Sistema cria registros:
   - AlmoxarifadoEstoque (1 para cada item/lote)
   - AlmoxarifadoMovimento (registro da entrada)
8. Atualiza estoque automaticamente
9. Redireciona com mensagem de sucesso
```

### 4.2 FLUXO DE SAÍDA

```
1. Usuário acessa /almoxarifado/saida
2. Seleciona funcionário e obra
3. Seleciona item → AJAX carrega estoque disponível
4. SE SERIALIZADO:
   - Mostra lista de itens disponíveis
   - Usuário seleciona itens específicos
5. SE CONSUMÍVEL:
   - Mostra quantidade disponível
   - Usuário informa quantidade a retirar
6. Sistema valida disponibilidade
7. Submete formulário → POST /almoxarifado/processar-saida
8. Sistema atualiza:
   - AlmoxarifadoEstoque (status ou quantidade)
   - Cria AlmoxarifadoMovimento
9. Vincula itens ao funcionário
10. Redireciona com sucesso
```

### 4.3 FLUXO DE DEVOLUÇÃO

```
1. Usuário acessa /almoxarifado/devolucao
2. Seleciona funcionário → AJAX busca itens em posse
3. GET /api/itens-funcionario/{id} retorna:
   - Itens serializados em posse
   - Consumíveis retornáveis com saldo
4. Usuário seleciona itens e quantidades
5. Submete formulário → POST /almoxarifado/processar-devolucao
6. Sistema processa:
   - Atualiza status/quantidade no estoque
   - Cria movimento de devolução
   - Remove vínculo com funcionário
7. Redireciona com sucesso
```

## 8. REGRAS DE NEGÓCIO

### 8.1 CONTROLE DE ESTOQUE

1. **Itens Serializados:**
   - Cada item tem número de série único
   - Status: DISPONÍVEL, EM_USO, MANUTENÇÃO, DESCARTADO
   - Sempre podem ser devolvidos
   - Rastreamento individual completo

2. **Itens Consumíveis:**
   - Controle por quantidade
   - Podem ter sobras devolvidas (se configurado)
   - Controle por lote/data de entrada
   - FIFO automático na saída

### 8.2 MOVIMENTAÇÕES

1. **Entrada:**
   - Sempre vinculada a uma obra (centro de custo)
   - Valor unitário obrigatório
   - Atualiza preço médio automaticamente

2. **Saída:**
   - Sempre vinculada a funcionário e obra
   - Verifica disponibilidade antes de processar
   - Atualiza status/quantidade automaticamente

3. **Devolução:**
   - Apenas itens que permitem devolução
   - Registra condição do item (para serializados)
   - Pode ser parcial (para consumíveis)

### 8.3 ALERTAS E NOTIFICAÇÕES

1. **Estoque Baixo:**
   - Alerta quando estoque <= estoque mínimo
   - Exibido no dashboard
   - Pode gerar notificação por email

2. **Validade de EPIs:**
   - Alerta 30 dias antes do vencimento
   - Controle automático por data de validade
   - Relatório específico

3. **Itens Sem Movimentação:**
   - Identifica itens parados há mais de 90 dias
   - Sugestão de remanejamento

## 9. VALIDAÇÕES E SEGURANÇA

### 9.1 VALIDAÇÕES DE ENTRADA

- **Multi-tenancy:** Todos os dados isolados por `admin_id`
- **Permissões:** Verificar se usuário pode acessar obra/funcionário
- **Integridade:** Validar FK antes de salvar
- **Unicidade:** Números de série únicos por item

### 9.2 VALIDAÇÕES DE NEGÓCIO

- **Estoque:** Não permitir saída maior que disponível
- **Datas:** Não permitir datas futuras em movimentações
- **Status:** Não permitir saída de itens em manutenção
- **Devolução:** Apenas itens que estão com o funcionário

### 9.3 AUDITORIA

- **Log de Movimentações:** Todos os movimentos registrados
- **Usuário Responsável:** Quem fez cada operação
- **Timestamp:** Data/hora de cada ação
- **Observações:** Campo livre para justificativas
